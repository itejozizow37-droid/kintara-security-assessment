# Client Configuration Exposure via /client-config.js and /api/version

**ID:** vuln-0008
**Severity:** MEDIUM
**Found:** 2026-07-12 23:05:37 UTC
**Target:** https://kintara.gg
**Endpoint:** /client-config.js
**Method:** GET
**CWE:** CWE-200
**CVSS:** 5.3

## Description

The client-side configuration file at /client-config.js exposes several security-relevant configuration values to all visitors without authentication. The file reveals: the read-only fanout server origin (https://fanout.kintara.gg), the client version SHA (49ef62cc53d001badfe71cec670dc172d2bd5142), the idle kick timeout in milliseconds (600000 = 10 minutes), and several feature flags including wallet picker, curated wallet list, server switch, and stale auto-reload capabilities. Additionally, the /api/version endpoint exposes the same git commit SHA.

## Impact

The exposed configuration values aid attacker reconnaissance: (1) The fanout server URL (fanout.kintara.gg) reveals the read-replica architecture and provides an additional attack surface; (2) The client version SHA can be used to identify specific code versions and match them to known vulnerabilities; (3) The idle kick timeout (10 minutes) tells attackers how long a session can remain idle before being disconnected, useful for session hijacking planning; (4) The feature flags reveal what capabilities are enabled, helping attackers focus on active attack vectors.

## Technical Analysis

The /client-config.js file is loaded by both the homepage and the game page. It sets global window variables with configuration values that should not be publicly exposed. The KINTARA_CLIENT_VERSION_SHA exposes a git commit hash, and the KINTARA_READ_FANOUT_ORIGIN reveals the fanout CDN architecture. The /api/version endpoint separately confirms the git SHA. The idle kick timeout (600000ms = 10 minutes) is a session management parameter that should not be exposed.

## Proof of Concept

1. Send GET https://kintara.gg/client-config.js
2. The response reveals: window.KINTARA_READ_FANOUT_ORIGIN, window.KINTARA_CLIENT_VERSION_SHA, window.KINTARA_IDLE_KICK_MS, and several feature flags.
3. GET https://kintara.gg/api/version returns: {"ok":true,"sha":"49ef62cc53d001badfe71cec670dc172d2bd5142"}

```
curl -sk --noproxy '*' -H "User-Agent: Mozilla/5.0" "https://kintara.gg/client-config.js"
curl -sk --noproxy '*' -H "User-Agent: Mozilla/5.0" "https://kintara.gg/api/version"
```

## Remediation

1. Remove sensitive configuration values from client-side JavaScript. Move configuration to server-side and fetch it via authenticated API calls.
2. Do not expose the git commit SHA in the client configuration or API response - this helps attackers identify specific code versions.
3. Do not expose the idle timeout value - this aids session management attacks.
4. If feature flags are needed client-side, fetch them via an authenticated API call rather than a static JS file.
