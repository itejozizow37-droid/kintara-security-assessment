# Unauthenticated Marketplace Data Exposure with Player Enumeration

**ID:** vuln-0003
**Severity:** MEDIUM
**Found:** 2026-07-12 23:00:03 UTC
**Target:** https://kintara.gg
**Endpoint:** /api/marketplace/listings
**Method:** GET
**CWE:** CWE-862
**CVSS:** 5.3

## Description

The endpoint GET /api/marketplace/listings on kintara.gg returns active marketplace listings without authentication. Each listing exposes sellerId (numeric player ID), sellerName (display name), itemType, quantity, priceGold, currency (gold or token), priceUsd, itemDurability, and createdAt timestamp. The endpoint also returns the total number of active listings (7,680+ at time of testing). The endpoint accepts limit and offset parameters, allowing complete enumeration of all listings and all sellers.

## Impact

An unauthenticated attacker can enumerate the complete marketplace by paginating through all listings using the limit and offset parameters. This exposes every active seller's player ID and display name, which can be cross-referenced with chat data and property ownership to build comprehensive player profiles. The priceUsd field also reveals the real-world USD value of each listing, providing economic intelligence about the game's marketplace and player trading patterns. The total listing count (7,680+) reveals the game's economic activity level.

## Technical Analysis

The /api/marketplace/listings endpoint returns paginated marketplace data. It accepts limit (default 20) and offset (default 0) parameters, and returns a JSON object with: ok, listings (array), total (count of all active listings), limit, offset, hasMore. Each listing object contains sellerId, sellerName, itemType, quantity, priceGold, currency, priceUsd, itemDurability, createdAt, reservedBy, reservedUntilMs. The endpoint does not validate the itemType filter parameter - it returns the same 20 listings regardless of the itemType value, but the limit/offset parameters work correctly for pagination. The total field reveals the complete marketplace size.

## Proof of Concept

1. Send GET https://kintara.gg/api/marketplace/listings (returns 20 listings + metadata)
2. The response includes: total (7680), limit (20), offset (0), hasMore (true)
3. Paginate with: GET /api/marketplace/listings?limit=100&offset=0, then offset=100, etc.
4. Each listing contains: id, sellerId, sellerName, itemType, quantity, priceGold, currency, priceUsd, createdAt, itemDurability
5. Example: {"id":886431,"sellerId":1816,"sellerName":"AVAstr","itemType":"mount_octopus","quantity":1,"priceGold":1,"currency":"token","priceUsd":7.99}

```
# Get first 100 listings
curl -sk --noproxy '*' \
  -H "User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/126.0.0.0 Safari/537.36" \
  "https://kintara.gg/api/marketplace/listings?limit=100&offset=0" | python3 -m json.tool | head -40

# Enumerate all sellers
for offset in 0 100 200 300 400 500; do
  curl -sk --noproxy '*' \
    -H "User-Agent: Mozilla/5.0" \
    "https://kintara.gg/api/marketplace/listings?limit=100&offset=$offset" | \
    python3 -c "import json,sys; d=json.load(sys.stdin); print(f'Offset {$offset}: {len(d.get(\"listings\",[]))} listings, total: {d.get(\"total\")}')"
done
```

## Remediation

1. Require authentication before returning marketplace listings, even for browsing.
2. Remove or hash the sellerId field for unauthenticated viewers - only show sellerName if needed for display.
3. Remove the priceUsd field from unauthenticated responses, as it reveals real-world economic value.
4. Implement rate limiting to prevent automated enumeration of all listings.
5. Consider returning only a subset of listings for unauthenticated users, requiring login to see the full marketplace.
